## 依赖关系

```
Phase 0（修 Bug）
    ↓
Phase 1（RAG + Agent + Skill）
    ├── RAG 集成（Day 3-4）
    ├── Agent 工具调用（Day 5-6）  ← 依赖 RAG
    └── Skill 标准化（Day 7-8）   ← 依赖 Agent 工具
    ↓
Phase 2（主 Agent + Wiki + 语音）
    ├── 主 Agent（Day 9-10）      ← 依赖 Agent 工具 + Skill
    ├── Wiki（Day 11-13）         ← 依赖 RAG
    └── 语音出题（Day 14-15）     ← 依赖 Skill + 档案
    ↓
Phase 3（工程化）
```

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Spring AI Function Calling 不稳定 | Agent 工具调用受阻 | 先用原生 HTTP 调用 LLM，再适配 Spring AI |
| RAG 检索效果差 | 知识增强效果不明显 | 调整检索参数、优化文档分块策略 |
| 主 Agent 工具选择不准确 | 用户体验差 | 优化工具描述、添加意图识别层 |
| 语音出题评估不准 | 题目质量低 | 多轮 Prompt 优化 + 人工审核 |
