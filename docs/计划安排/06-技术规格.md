## 一、新增数据模型

### 1.1 NvcWikiEntity（Wiki 条目）

```sql
CREATE TABLE nvc_wiki (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    title VARCHAR(255) NOT NULL,
    content TEXT,
    category VARCHAR(50) NOT NULL,  -- PRACTICE_NOTE / REAL_SCENARIO / LEARNING_SUMMARY
    tags VARCHAR(500),
    source_type VARCHAR(50) NOT NULL,  -- AUTO_GENERATED / MANUAL / AI_ASSISTED
    source_session_id BIGINT,
    file_path VARCHAR(500),
    vector_id VARCHAR(255),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_nvc_wiki_user ON nvc_wiki(user_id);
CREATE INDEX idx_nvc_wiki_category ON nvc_wiki(user_id, category);
```

### 1.2 NvcAssistantMessageEntity（主 Agent 对话历史）

```sql
CREATE TABLE nvc_assistant_message (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    role VARCHAR(20) NOT NULL,  -- USER / ASSISTANT
    content TEXT,
    tool_calls_json TEXT,  -- 工具调用链 JSON
    created_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_nvc_assistant_msg_user ON nvc_assistant_message(user_id);
```

### 1.3 KnowledgeBaseEntity 变更

```sql
ALTER TABLE knowledge_bases ADD COLUMN type VARCHAR(50);
-- NVC_THEORY / SPEECH_TEMPLATE / EMOTION_VOCAB / USER_CASE / PERSONAL_WIKI
```

### 1.4 NvcVoiceQuizSessionEntity（语音出题会话）

```sql
CREATE TABLE nvc_voice_quiz_session (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    status VARCHAR(20) NOT NULL,  -- IN_PROGRESS / COMPLETED
    questions_json TEXT,  -- 题目列表 JSON
    current_question_index INT DEFAULT 0,
    results_json TEXT,  -- 结果 JSON
    created_at TIMESTAMP DEFAULT NOW(),
    completed_at TIMESTAMP
);
```

## 二、新增 API 端点

### 2.1 主 Agent

| 方法 | URL | 功能 | 请求体 | 响应 |
|------|-----|------|--------|------|
| POST | /api/nvc/assistant/chat | 非流式对话 | `{content: string}` | `{reply: string, toolCalls: [...]}` |
| POST | /api/nvc/assistant/chat/stream | 流式对话 | `{content: string}` | SSE events |
| GET | /api/nvc/assistant/history | 对话历史 | `?userId=&limit=` | `[{role, content, toolCalls}]` |

### 2.2 Wiki

| 方法 | URL | 功能 | 请求体 | 响应 |
|------|-----|------|--------|------|
| POST | /api/nvc/wiki | 创建 Wiki | `{title, content, category, tags}` | WikiResponse |
| GET | /api/nvc/wiki | 列表 | `?userId=&category=&tag=&page=&size=` | Page<WikiResponse> |
| GET | /api/nvc/wiki/{id} | 详情 | - | WikiResponse |
| PUT | /api/nvc/wiki/{id} | 更新 | `{title, content, tags}` | WikiResponse |
| DELETE | /api/nvc/wiki/{id} | 删除 | - | void |
| GET | /api/nvc/wiki/search | 语义搜索 | `?userId=&q=&topK=` | List<WikiResponse> |

### 2.3 语音出题

| 方法 | URL | 功能 | 请求体 | 响应 |
|------|-----|------|--------|------|
| POST | /api/nvc/voice-quiz/sessions | 创建会话 | `{userId, questionCount}` | VoiceQuizSession |
| GET | /api/nvc/voice-quiz/sessions/{id}/question | 当前题目 | - | `{question, index, total}` |
| POST | /api/nvc/voice-quiz/sessions/{id}/answer | 提交答案 | `{audioData: base64}` | `{score, feedback}` |
| GET | /api/nvc/voice-quiz/sessions/{id}/result | 结果汇总 | - | `{totalScore, details: [...]}` |

## 五、技术决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| Agent 工具调用 | Spring AI Function Calling | 与技术栈一致，原生支持 |
| Wiki 存储 | 文件系统 + DB + 向量 | 文件直观可编辑，DB 管理元数据，向量支持语义检索 |
| RAG 检索 | 混合检索（向量 + 关键词） | 提高召回率 |
| Skill 体系 | Java 泛型接口 + Spring 注解 | 类型安全，易于扩展 |
| 主 Agent 思考链 | Prompt Engineering | 灵活，易调整 |
| 练习 Agent 编排 | ModeRouter 策略模式 | 已有设计，扩展性好 |
| 向量数据库 | pgvector | 已有基础设施，无需引入新组件 |
| Embedding 模型 | DashScope text-embedding-v3 | 与现有 LLM Provider 统一，1024 维 |
| 前端状态管理 | React hooks + API 缓存 | 简单直接 |
| 个性化权重 | 基于用户档案动态调整 | 实现简单，效果明显 |
| 知识库隔离 | 逻辑隔离（type 字段） | 比物理隔离更灵活 |
| Wiki 文件格式 | Markdown | 通用、可编辑、可渲染 |
| 评估输出 | BeanOutputConverter + JSON | 结构化、可解析、可重试 |
| 流式对话 | SSE（Server-Sent Events） | 浏览器原生支持、实现简单 |
| 语音通信 | WebSocket | 全双工、实时性好 |
