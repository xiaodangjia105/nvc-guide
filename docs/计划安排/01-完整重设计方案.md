## 1. 项目定位

基于 Spring Boot 4.0 + Spring AI 2.0 + React 18 的 AI 驱动非暴力沟通学习平台。

**核心卖点：**
- 多 Agent 编排架构（10 种专职 Agent，策略路由）
- Agent 工具调用框架（Spring AI Function Calling）
- 个人知识 Wiki（自动沉淀 + 语义检索）
- RAG 增强对话（多知识库 + 个性化权重）
- 可扩展 Skill 体系（评估、场景生成、对话引导）

## 3. Phase 0：修 Bug + 稳定现有（2 天）

### 3.1 对话流程修复

**问题：** 场景模式下 Agent 角色感弱，练习流程不完整。

**方案：**
- 优化 `ScenarioRouter` 路由逻辑，基于对话轮次和状态智能切换 Agent
- 添加 `NvcSessionPhase.EVALUATING` 状态，练习结束前先评估再完成
- 场景模式下 AI 回复增加 NVC 教练的专业感（Prompt 优化）

**修改文件：**
- `router/ScenarioRouter.java` — 路由逻辑优化
- `model/NvcSessionPhase.java` — 添加 EVALUATING 状态
- `service/NvcPracticeDialogueService.java` — 流程状态机完善

### 3.2 评估优化

**问题：** 评估反馈太笼统，评估和对话脱节。

**方案：**
- 优化评估 Prompt，要求输出具体、可操作的改进建议
- 评估结果包含"下一步练习建议"
- 评估和对话流程融合（评估 Skill 可被 Agent 调用）

**修改文件：**
- `resources/prompts/nvc-evaluation-system.st` — Prompt 优化
- `resources/prompts/nvc-evaluation-summary-system.st` — Prompt 优化
- `service/NvcEvaluationService.java` — 增加改进建议字段

### 3.3 其他修复

- 语音模块改名：`voiceinterview` → `nvcvoice`
- 练习结束时自动触发档案更新
- 修复前端已知显示问题

## 4. Phase 1：RAG + Agent 升级（7 天）

### 4.1 RAG 知识库集成

**当前状态：** KnowledgeBase 模块已存在（文档上传/分块/向量化），但未与 NVC 练习集成。

**设计方案：**

#### 4.1.1 多知识库管理

```java
// KnowledgeBaseEntity 新增 type 字段
public enum KnowledgeBaseType {
    NVC_THEORY,      // NVC 理论知识
    SPEECH_TEMPLATE, // 话术模板
    EMOTION_VOCAB,   // 情绪词汇
    USER_CASE,       // 用户案例
    PERSONAL_WIKI    // 个人 Wiki（按用户隔离）
}
```

#### 4.1.2 NvcRagService

```java
@Service
public class NvcRagService {

    /**
     * 统一 RAG 检索
     * @param query 检索查询
     * @param userId 用户ID（用于个性化和 Wiki 隔离）
     * @param scenarioType 场景类型（用于过滤）
     * @param knowledgeTypes 检索的知识库类型
     * @param topK 返回结果数量
     */
    List<RagResult> retrieve(String query, Long userId,
                             ScenarioType scenarioType,
                             List<KnowledgeBaseType> knowledgeTypes,
                             int topK);

    /**
     * 个性化检索（基于用户薄弱要素加权）
     */
    List<RagResult> retrievePersonalized(String query, Long userId,
                                          NvcPracticeStep weakElement);
}
```

**检索策略：**
1. 向量检索（pgvector COSINE 相似度）
2. 关键词检索（PostgreSQL full-text search）
3. 混合排序（向量分数 × 0.7 + 关键词分数 × 0.3）
4. 个性化加权（用户薄弱要素相关内容 × 1.5）

#### 4.1.3 Agent 对话集成 RAG

```java
// NvcAgentChatService.buildMessages() 中增加
if (context.getScenarioDescription() != null) {
    // 自动检索相关知识
    List<RagResult> ragResults = ragService.retrieve(
        context.getScenarioDescription(),
        context.getSession().getUserId(),
        scenarioType,
        List.of(KnowledgeBaseType.NVC_THEORY,
                KnowledgeBaseType.SPEECH_TEMPLATE),
        3
    );
    systemPrompt += "\n\n[参考资料]\n" + formatRagResults(ragResults);
}
```

#### 4.2.3 NvcToolRegistry

```java
@Service
public class NvcToolRegistry {
    private final Map<String, NvcTool> tools;

    public NvcToolRegistry(List<NvcTool> toolList) {
        this.tools = toolList.stream()
            .collect(Collectors.toMap(NvcTool::name, t -> t));
    }

    public NvcTool getTool(String name) { return tools.get(name); }
    public List<NvcTool> getAllTools() { return List.copyOf(tools.values()); }

    /** 转换为 Spring AI Function Callback */
    public List<FunctionCallback> toFunctionCallbacks() { ... }
}
```

#### 4.2.4 Spring AI Function Calling 集成

```java
// NvcAgentChatService 升级
public String chat(NvcAgentConfigEntity config, PracticeContext context,
                   String userMessage, Map<String, String> promptVariables) {
    ChatClient client = getChatClient(config);
    List<Message> messages = buildMessages(config, context, userMessage, promptVariables);

    return client.prompt()
        .options(buildChatOptions(config))
        .functions(toolRegistry.toFunctionCallbacks()) // 注入工具
        .messages(messages)
        .call()
        .content();
}
```

## 7. 技术决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| Agent 工具调用 | Spring AI Function Calling | 与技术栈一致，原生支持 |
| Wiki 存储 | 文件系统 + DB + 向量 | 文件直观可编辑，DB 管理元数据，向量支持语义检索 |
| RAG 检索 | 混合检索（向量 + 关键词） | 提高召回率 |
| Skill 体系 | Java 接口 + Spring 注解 | 类型安全，易于扩展 |
| 主 Agent 思考链 | Prompt Engineering | 灵活，易调整 |
| 练习 Agent 编排 | ModeRouter 策略模式 | 已有设计，扩展性好 |
| 向量数据库 | pgvector | 已有基础设施 |
| 前端状态管理 | React hooks + API 缓存 | 简单直接 |
