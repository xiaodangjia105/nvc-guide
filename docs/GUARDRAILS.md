# 改动安全区 — loop 护栏

> 自主 loop 的边界。区内可自行改并自测；区外触碰前必须停下向人确认。规则优先于"任务完成"。

## 🔴 高危区（碰前必须人工确认）
- **生产配置与密钥**：任何 profile 配置、加密配置项、第三方地址/凭据。不改、不解密、不打印明文。
- **数据库 schema**：实体字段/表结构改动会经 Hibernate `ddl-auto: update` **自动改生产表**。改实体即改库，必须先确认。
- **Agent 配置与 Prompt**：NVC Agent 的 system prompt、评估标准、四要素权重。改动影响所有用户的练习体验。
- **LLM Provider 配置**：模型选择、API Key、超时参数。改动影响全局 AI 服务可用性。
- **状态机与码值**：练习会话状态、评估等级、错误码、Agent 场景枚举。改动牵连前端和业务流程。
- **第三方对接**：ASR/TTS 服务、向量数据库、文件存储的协议与配置。
- **鉴权与权限**：令牌、会话、权限注解的信任边界。

## 🟢 安全区（可自行改 + 自测）
- 单个页面/组件内部实现、样式、文案。
- 不改契约的纯前端交互优化。
- 新增不影响既有流程的内部工具方法。
- 补测试、补注释（按 SKILL.md，注释只写"为什么"）。
- Prompt 模板的措辞微调（不改结构和评估逻辑）。

## 🟡 灰区（可做，但需在结果里显式说明改了什么、影响面）
- 跨多文件的重构、新增接口、依赖调整。
- 触及通用组件（Layout/AuthForm 等被多页面复用）。
- 新增或修改 Redis Stream 消费者/生产者。
- 修改 Agent 调度逻辑（NvcAgentOrchestrator）。

## 通用红线
- 不 `--no-verify`、不跳过校验绕过失败。
- 不为"让报错消失"而删数据/删分支/改测试预期。
- 区分不清归属时，按高危同理 → 先问。
