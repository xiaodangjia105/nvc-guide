# 系统边界

> 前后端职责划分与通信方式。决定"哪些规则在前端做、哪些必须在后端做"。
> 本系统为单体仓库：后端（Spring Boot）+ 前端（React SPA），经 HTTP 集成。

## 职责划分

| 维度 | 前端（React + TS） | 后端（Spring Boot） |
|------|------|------|
| 展示与交互 | 页面渲染、表单体验、导航、主题 | — |
| 路由 | 前端路由（React Router） | — |
| 业务规则 | 仅做体验层校验 | Agent 调度、评估逻辑、会话状态机全部规则 |
| AI 服务 | — | LLM 调用、Prompt 模板、结构化输出、向量搜索 |
| 数据一致性与持久化 | — | 事务、JPA 实体、Redis 缓存、异步评估 |

## 必须在后端兜底的规则
> 前端的表单校验、状态判断都只是体验优化，后端必须独立校验。

- **Agent 调度**：下一个 Agent 的选择、上下文组装、对话历史管理只在后端。
- **评估逻辑**：四要素评分、表达质量判定只在后端，前端仅展示结果。
- **会话状态**：练习会话的生命周期（进行中→已完成→已评估）由后端管理。
- **AI 服务调用**：LLM Provider 选择、重试、超时处理由后端统一管理。

## 前后端通信方式
- **同步接口**：HTTP REST，统一前缀 `/api/nvc/*`。响应/错误码见 `docs/contracts/api.md`。
- **流式响应**：SSE（Server-Sent Events）用于 AI 对话的流式输出。
- **WebSocket**：语音练习模块使用 WebSocket 实时通信（ASR/TTS）。
- **文件上传**：`multipart/form-data`，用于知识库文档上传。
