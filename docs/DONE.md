# 完成标准（DoD）

> loop 的收口判据。任一项不满足，任务即未完成，不得宣称成功。

## 通用判据

- [ ] 改动在独立分支上完成（`feat/*` / `fix/*` / `refactor/*`），不在 master 上直接改。
- [ ] 改动落在 `GUARDRAILS.md` 安全区；触及高危/灰区的已按规则先确认或显式说明影响面。
- [ ] 后端编译通过（`./gradlew build`），前端构建通过（`pnpm build`）。
- [ ] 受影响的核心流程已手测或说明为何无法测。
- [ ] 未新增临时绕过：无 `--no-verify`、无为消除报错而改测试预期/删数据。
- [ ] 契约未被悄悄改动；如确需改，已同步 `docs/contracts/` 与前后端两侧。
- [ ] 触及接口形态（字段/错误码）时，已用代码核对 `docs/contracts/`。

## 文档回写判据

> loop 的"沉淀认知"环节。完成任务后按需回写，但遵守 SKILL.md（写能力不写路径）。

- [ ] 新出现的非显而易见约定 → 写入对应服务 `AGENTS.md` 内部约定。
- [ ] 一次性踩坑/取舍 → 记入 `docs/decisions/`。
- [ ] 新领域术语 → 补入 `GLOSSARY.md`。
- [ ] 确认了原本"待确认"的事实 → 回填原占位，删除过时标注。

## 诚实性判据

- [ ] 无法验证的部分已显式声明，未用"应该可以"掩盖。
- [ ] 报告区分了"已验证"与"按约定推断"。
