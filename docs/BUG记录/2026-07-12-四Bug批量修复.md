## BUG1：无法从练习记录继续进行中的对话

### 现象
在练习历史页面点击一个"进行中"的会话，页面一直转圈加载，无法继续对话。

### 根因

**前端**：`NvcHistoryPage.tsx` 第 111 行对所有状态的会话统一跳转到报告页：

```tsx
onClick={() => navigate(`/nvc/history/${session.id}/report`)}
```

没有区分会话阶段。IN_PROGRESS 的会话被带到报告页。

**后端**：`NvcReportService.generateReport()` 发现没有 FINAL 评估时，会同步调用 `evaluateFinal()`（LLM API 调用，最长 300 秒/次，最多重试 2 次）。前端 axios 超时 60 秒，页面一直转圈。

### 修复

**前端**（`NvcHistoryPage.tsx`）：根据 `currentPhase` 分别导航：

```tsx
onClick={() => {
  const phase = session.currentPhase;
  if (phase === 'CREATED' || phase === 'IN_PROGRESS' || phase === 'PAUSED') {
    navigate(`/nvc/practice/${session.id}`);
  } else {
    navigate(`/nvc/history/${session.id}/report`);
  }
}}
```

**后端**（`NvcReportService.java`）：对非 COMPLETED/EVALUATED 会话直接返回默认报告，不触发 LLM：

```java
if (session.getCurrentPhase() != NvcSessionPhase.COMPLETED
    && session.getCurrentPhase() != NvcSessionPhase.EVALUATED) {
    return buildDefaultReport(session, messages);
}
```

### 涉及文件
- `frontend/src/pages/NvcHistoryPage.tsx` — 第 111 行
- `app/.../service/NvcReportService.java` — 第 42-52 行

## BUG2：场景对话模式 AI 回复包含原始 JSON

### 现象
场景驱动模式下，AI 第一条回复中包含原始 JSON 结构：

```
是啊
{ "scenario_title": "功劳归属的沟通", "background_description": "...", ... }
```

### 根因（三层问题）

**① 场景信息重复注入**

`ScenarioRouter` 将场景描述注入到 `promptVariables` 的 `scenario_context`，而 `buildMessages` 又通过 `[练习场景]` 注入了同样的内容。LLM 收到两次结构化信息后倾向于"回显"为 JSON 格式。

**② 流式路径未经清理**

`sendMessageStream` 中 token 原样发给前端，`AiResponseCleaner.clean()` 只在 `doOnComplete` 中对存 DB 的副本执行。用户看到的是原始 LLM 输出。

**③ JSON 清理正则不支持混合内容**

`AiResponseCleaner` 的正则 `^\s*\{[\s\S]*?\}\s*$` 只匹配整段是 JSON 的情况，对"文本 + JSON"的混合内容（如"是啊\n{ ... }"）无法匹配。

### 修复

1. **去掉重复注入**（`ScenarioRouter.java`）：删除 `scenario_context` 变量注入
2. **增强清理器**（`AiResponseCleaner.java`）：新增 `removeInlineJson` 方法，用大括号计数处理嵌套 JSON
3. **前端同步增强**（`NvcChatPanel.tsx`）：前端 `cleanAiResponse` 也添加 `removeInlineJson`
4. **强化 prompt**（`data-nvc-agent-config.sql`）：SCENARIO_GENERATOR prompt 增加"绝对不要输出 JSON"的明确指令和正反示例

### 涉及文件
- `app/.../router/ScenarioRouter.java` — 去掉重复注入
- `app/.../util/AiResponseCleaner.java` — 新增 `removeInlineJson`
- `frontend/src/components/nvc/NvcChatPanel.tsx` — 前端清理增强
- `app/src/main/resources/data-nvc-agent-config.sql` — SCENARIO_GENERATOR prompt 强化

## BUG3：场景库 AI 生成后刷新看不见新场景

### 现象
在场景库页面新建场景并点击"AI 生成"，加载完成后刷新页面，新场景消失了。有时 spinner 转完后什么都没发生。

### 根因（三层问题）

**① LLM 返回带 markdown 的 JSON，后端解析失败**

`nvc-scenario-generate-system.st` 说"以JSON格式返回"但没有禁止 markdown 代码块。LLM 可能返回：
```json
{"title": "...", ...}
```
`objectMapper.readTree()` 无法解析带 markdown 的内容 → 抛出 `BusinessException(AI_SERVICE_ERROR)`

**② 默认查询只返回系统场景**

`NvcScenarioService.queryScenarios()` 无筛选时：

```java
return scenarioRepository.findByIsSystemTrueOrderByUsageCountDesc();
```

只返回 `isSystem = true` 的场景。AI 生成的场景 `isSystem = false`，在默认视图中被过滤掉。

**③ 前端静默吞掉错误**

`handleGenerate` 的 catch 块只 `console.error`，用户看不到任何错误提示。

### 修复

**Prompt**（`nvc-scenario-generate-system.st`）：增加严格输出要求，禁止 markdown 代码块。

**后端**（`NvcScenarioService.java`）：
- 新增 `extractJson()` 方法，支持从 markdown 代码块、混合内容中提取纯 JSON
- 默认查询改为 `findAllByOrderByUsageCountDesc()`

**前端**（`NvcScenarioLibraryPage.tsx`）：
- catch 块增加 `alert` 提示用户
- 生成成功后调用 `loadScenarios()` 从服务器重新拉取完整列表

### 涉及文件
- `app/src/main/resources/prompts/nvc-scenario-generate-system.st` — prompt 强化
- `app/.../service/NvcScenarioService.java` — extractJson + 默认查询
- `app/.../repository/NvcScenarioRepository.java` — 新增方法
- `frontend/src/pages/NvcScenarioLibraryPage.tsx` — 错误提示 + 列表刷新

## BUG4："你的表达梳理"面板不会更新已填充内容

### 现象
对话页面右侧的"你的表达梳理"面板，当用户后续描述得更清晰时，已经填充过的观察/感受/需求/请求不会被更新。

### 根因

`NvcSummaryService.mergeField()` 的逻辑是"只写空字段"：

```java
if (existing == null || existing.isBlank()) {
    setter.accept(newValue);
}
// 已有值 → 不覆盖
```

字段一旦被填充，后续 LLM 返回的更新值被直接丢弃。

### 修复

1. **`mergeField` 改为始终更新**：去掉 `existing == null || existing.isBlank()` 判断，直接 `setter.accept(newValue)`
2. **Prompt 同步优化**：明确要求 LLM 返回每个要素的"最终完整值"（综合已有摘要和新内容），而非只返回新增部分

### 涉及文件
- `app/.../service/NvcSummaryService.java` — `mergeField` 方法 + SUMMARY_PROMPT
