### 2.2 分层规则（严格自上而下）

```
Controller → Service → Repository
     │           │          │
     │           │          └── JPA/MyBatis，纯数据访问
     │           └── 业务逻辑，事务控制，外部 API 调用
     └── 路由 + 委派，禁止业务逻辑
```

**关键约束**：
- Controller 只做路由和委派，不写业务逻辑
- Service 层控制事务，事务内禁止调用 LLM 等外部 API
- Entity 永远不直接返回给前端，必须转 DTO
- 业务错误只抛 `BusinessException(ErrorCode.XXX, message)`

## 三、核心模块深度解析

### 3.1 NVC 练习模块（最复杂）

这是整个系统的核心，负责 NVC 练习的全生命周期。

#### 核心类关系图

```
NvcPracticeController
    │
    ├── NvcPracticeDialogueService    # 对话服务（编排层）
    │       │
    │       ├── NvcAgentOrchestrator   # Agent 调度中心（大脑）
    │       │       │
    │       │       ├── ModeRouter          # 路由策略接口
    │       │       │   ├── FreeDialogRouter    # 自由对话路由
    │       │       │   ├── ScenarioRouter      # 场景驱动路由
    │       │       │   └── StructuredRouter    # 结构化四步路由
    │       │       │
    │       │       ├── NvcAgentChatService  # LLM 对话执行
    │       │       │       │
    │       │       │       └── LlmProviderRegistry  # LLM 客户端管理
    │       │       │
    │       │       ├── NvcRagService        # RAG 知识检索
    │       │       └── NvcProfileService    # 用户档案
    │       │
    │       ├── NvcEvaluationService   # 评估服务
    │       └── NvcSummaryService      # 摘要服务
    │
    ├── NvcPracticeSessionService    # 会话管理
    └── NvcStructuredPracticeService # 结构化四步管理
```

#### Agent 场景（11 种专职 Agent）

```java
public enum NvcAgentScene {
    SCENARIO_GENERATOR,          // 场景生成器
    DIALOGUE_GUIDE,              // 对话引导者
    DIFFICULT_PARTNER,           // 难缠对手
    STEP_OBSERVE_COACH,          // 观察步骤教练
    STEP_FEELING_COACH,          // 感受步骤教练
    STEP_NEED_COACH,             // 需求步骤教练
    STEP_REQUEST_COACH,          // 请求步骤教练
    NVC_EXPRESSION_EVALUATOR,    // NVC 表达评估器
    EMPATHY_COACH,               // 共情教练
    NVC_KNOWLEDGE_ADVISOR,       // NVC 知识顾问
    PROFILE_ANALYZER             // 档案分析师
}
```

每个 Agent 有独立的：
- System Prompt（存 DB，可热更新）
- Temperature、MaxTokens 等参数
- 可用工具列表（按场景过滤）

#### 三种练习模式

| 模式 | Router | 特点 |
|---|---|---|
| **FREE_DIALOG** | FreeDialogRouter | 自由对话，Agent 自由发挥 |
| **SCENARIO** | ScenarioRouter | 场景驱动，先选场景再练习 |
| **STRUCTURED_FOUR_STEP** | StructuredRouter | 观察→感受→需求→请求，逐步推进 |

### 4.2 文本练习 — 发送消息（流式 SSE）

流式和非流式的核心逻辑相同，区别在于 LLM 调用方式和事件流组装。

```
前端                              后端
  │                                 │
  │  POST /api/nvc/practice/sessions/{id}/messages/stream
  │  Accept: text/event-stream
  │ ──────────────────────────────► │
  │                                 │
  │  （步骤 1-5 同非流式）            │
  │                                 │
  │                    ┌────────────▼────────────────────┐
  │                    │ 步骤 6: 流式执行 Agent 对话        │
  │                    │   orchestrator.executeAgentStream │
  │                    │   → 返回 Flux<String> (token 流)  │
  │                    └────────────┬────────────────────┘
  │                                 │
  │                    ┌────────────▼────────────────────┐
  │                    │ 步骤 7: 组装 SSE 事件流            │
  │                    │                                 │
  │                    │  event: metadata                  │
  │                    │  data: {scene, reason, action}    │
  │                    │                                 │
  │                    │  event: message                   │
  │                    │  data: "我"                       │
  │                    │                                 │
  │                    │  event: message                   │
  │                    │  data: "理解"                     │
  │                    │                                 │
  │                    │  event: message                   │
  │                    │  data: "你的感受..."               │
  │                    │                                 │
  │                    │  event: done                      │
  │                    │  data: {"length": 156}            │
  │                    └────────────┬────────────────────┘
  │                                 │
  │  ◄─── SSE stream ────────────── │
  │                                 │
  │                    ┌────────────▼────────────────────┐
  │                    │ doOnComplete: 流结束后             │
  │                    │   ├─ 保存 AI 回复到 DB            │
  │                    │   ├─ 异步触发摘要更新              │
  │                    │   └─ 检查步骤推进                  │
  │                    └─────────────────────────────────┘
```

**SSE 事件格式**：
```
event: metadata
data: {"scene":"DIALOGUE_GUIDE","reason":"用户首次发言","action":null,"currentStep":null}

event: message
data: 我理解你感到沮丧

event: message
data: ，这很不容易

event: done
data: {"length":42}
```

### 4.3 场景驱动模式链路

```
1. 用户选择场景
   └─ GET /api/nvc/scenarios → 场景列表

2. 创建练习会话（指定场景）
   └─ POST /api/nvc/practice/sessions
      { practiceMode: "SCENARIO", scenarioId: 42 }

3. 发送消息
   └─ 同 4.1/4.2 链路，但：
      ├─ PracticeContext 包含 scenarioDescription
      ├─ System Prompt 注入场景信息
      └─ ScenarioRouter 路由逻辑：
         ├─ 首次 → SCENARIO_GENERATOR（生成场景开场白）
         └─ 后续 → DIALOGUE_GUIDE 或 DIFFICULT_PARTNER
```

### 4.4 结构化四步练习链路

```
1. 创建练习会话
   └─ POST /api/nvc/practice/sessions
      { practiceMode: "STRUCTURED_FOUR_STEP" }
      → currentStep = OBSERVE（第一步）

2. 发送消息 → 同 4.1 链路，但：
   ├─ StructuredRouter 路由：
   │   ├─ OBSERVE → STEP_OBSERVE_COACH
   │   ├─ FEELING → STEP_FEELING_COACH
   │   ├─ NEED → STEP_NEED_COACH
   │   └─ REQUEST → STEP_REQUEST_COACH
   │
   └─ Agent 回复中包含 action: "STEP_ADVANCE" 时：
       → 自动推进到下一步

3. 手动推进步骤
   └─ POST /api/nvc/practice/sessions/{id}/advance-step

4. 查看步骤进度
   └─ GET /api/nvc/practice/sessions/{id}/step-progress
```

### 4.5 用户档案模块链路

```
1. 查看档案
   └─ GET /api/nvc/profile?userId=1
      → NvcProfileController
      → NvcProfileService.getOrCreateProfile()
      → 返回 UserProfileDTO

2. 更新档案
   └─ PUT /api/nvc/profile?userId=1
      { nvcLevel: "INTERMEDIATE", communicationStyle: "ANALYTICAL" }
      → NvcProfileService.updateProfile()

3. 能力雷达图
   └─ GET /api/nvc/profile/ability-radar?userId=1
      → NvcProfileService.getAbilityRadar()
      → { observation: 75, feeling: 60, need: 80, request: 55 }

4. 能力趋势
   └─ GET /api/nvc/profile/ability-trends?userId=1
      → NvcProfileService.getAbilityTrends()
      → [{ date: "2026-07-01", overall: 65 }, ...]

5. 沟通分析
   └─ POST /api/nvc/profile/communication-records/analyze?userId=1
      { text: "我觉得你总是不听我说话..." }
      → NvcCommunicationAnalysisService.analyze()
      → AI 分析沟通模式，返回分析结果
```

### 4.6 语音练习模块链路

```
1. 创建语音会话
   └─ POST /api/nvc/voice/sessions
      { userId, scenarioId, difficulty }
      → NvcVoiceService.createSession()

2. 建立 WebSocket 连接
   └─ ws://localhost:8080/ws/nvc/voice?sessionId=123
      → NvcVoiceWebSocketHandler

3. 语音交互流程（WebSocket 消息）：
   ┌─────────────────────────────────────────────────────┐
   │  浏览器                    后端                       │
   │    │                        │                        │
   │    │ ── [binary: 音频] ───► │                        │
   │    │                        │ ┌──────────────────┐   │
   │    │                        │ │ VoicePipeline     │   │
```

## 五、核心设计模式解析

### 5.1 LlmProviderRegistry — LLM 客户端注册中心

这是整个 AI 能力的基石，管理所有 LLM 提供商的 ChatClient。

```
LlmProviderRegistry
│
├── clientCache: Map<String, ChatClient>
│   ├── "dashscope" → ChatClient (带工具、带记忆)
│   ├── "dashscope:plain" → ChatClient (无工具，用于评估)
│   └── "dashscope:voice" → ChatClient (语音专用)
│
├── embeddingModelCache: Map<String, EmbeddingModel>
│   └── "dashscope" → EmbeddingModel (向量化)
│
├── getChatClient(providerId)     → 带工具+记忆的客户端
├── getPlainChatClient(providerId) → 无工具客户端（评估、反思）
├── getVoiceChatClient(providerId) → 语音专用客户端
└── getEmbeddingModel(providerId)  → 向量模型
```

**客户端创建流程**：
```
loadProviderOrThrow(providerId)
    → 从 DB 读取配置（含解密 API Key）
    → 构建 OpenAiApi（适配各厂商 API 路径）
    → 构建 OpenAiChatModel
    → 添加 Advisors（ToolCall + Memory + Logger + SafeGuard）
    → 构建 ChatClient
    → 缓存到 clientCache
```

### 5.2 ModeRouter — 策略模式

```
ModeRouter (接口)
    │
    ├── FreeDialogRouter.route(context)
    │   └─ 简单路由：始终返回 DIALOGUE_GUIDE
    │
    ├── ScenarioRouter.route(context)
    │   ├─ 首次发言 → SCENARIO_GENERATOR
    │   ├─ 对话轮次少 → DIALOGUE_GUIDE
    │   └─ 对话轮次多 → DIFFICULT_PARTNER
    │
    └── StructuredRouter.route(context)
        ├─ OBSERVE → STEP_OBSERVE_COACH
        ├─ FEELING → STEP_FEELING_COACH
        ├─ NEED → STEP_NEED_COACH
        └─ REQUEST → STEP_REQUEST_COACH
```

### 5.3 NvcTool — Agent 工具框架

```
NvcTool (接口)
    │
    ├── name()           → 工具名称
    ├── description()    → 工具描述（给 LLM 看）
    ├── inputSchema()    → JSON Schema（参数定义）
    └── execute(input, context) → 执行逻辑
    │
    ├── 实现类：
    │   ├── RagSearchTool         → RAG 知识检索
    │   ├── ScenarioSearchTool    → 场景搜索
    │   ├── ScenarioGenerateTool  → 场景生成
    │   ├── ProfileQueryTool      → 用户档案查询
    │   ├── ProfileUpdateTool     → 用户档案更新
    │   ├── EvaluateNvcTool       → NVC 评估
    │   ├── DashboardQueryTool    → 仪表盘查询
    │   ├── PracticeStartTool     → 练习启动
    │   ├── WikiSearchTool        → Wiki 搜索
    │   └── WikiWriteTool         → Wiki 写入
    │
    └── NvcToolRegistry（注册中心）
        ├─ Spring 自动注入所有 NvcTool 实现
        ├─ toFunctionCallbacks() → 转换为 Spring AI FunctionToolCallback
        └─ 按场景过滤可用工具（NvcToolSceneMapping）
```

### 5.4 Redis Stream — 异步处理框架

```
AbstractStreamProducer
    │
    ├── send(streamKey, message)
    │   → XADD streamKey * field1 value1 field2 value2
    │
    ├── 实现类：
    │   ├── VectorizeStreamProducer     → 发送向量化任务
    │   ├── NvcEvaluateStreamProducer   → 发送评估任务
    │   └── NvcVoiceEvaluateStreamProducer → 发送语音评估任务

AbstractStreamConsumer
    │
    ├── @PostConstruct start()
    │   → 启动线程循环 XREAD
    │
    ├── consume(message)
    │   → 子类实现具体处理逻辑
    │
    ├── 实现类：
    │   ├── VectorizeStreamConsumer     → 执行向量化
    │   ├── NvcEvaluateStreamConsumer   → 执行评估
    │   └── NvcVoiceEvaluateStreamConsumer → 执行语音评估
```

## 十一、学习要点

### 11.1 架构亮点

1. **LlmProviderRegistry 统一管理**：所有 LLM 调用通过一个注册中心，支持多提供商切换、缓存、加密
2. **ModeRouter 策略模式**：练习模式解耦，新增模式只需实现接口
3. **NvcTool 工具框架**：Agent 工具标准化，Spring 自动注入 + 场景过滤
4. **Redis Stream 异步**：轻量级消息队列，避免引入 Kafka/RabbitMQ
5. **pgvector 向量存储**：复用 PostgreSQL，无需额外向量数据库

### 11.2 设计取舍

| 决策 | 选择 | 原因 |
|---|---|---|
| 向量数据库 | pgvector | 复用已有 PostgreSQL，运维简单 |
| 消息队列 | Redis Stream | 轻量级，避免引入 Kafka |
| LLM 客户端 | Spring AI ChatClient | 统一抽象，支持多提供商 |
| 异步评估 | Redis Stream + Consumer | 不阻塞对话，最终一致性 |
| 前端状态 | 无全局状态管理 | 页面独立，API 驱动 |

### 11.3 扩展点

- **新增 LLM 提供商**：在 DB 添加配置，LlmProviderRegistry 自动识别
- **新增练习模式**：实现 ModeRouter 接口，注册到 NvcAgentOrchestrator
- **新增 Agent 工具**：实现 NvcTool 接口，Spring 自动注入到 NvcToolRegistry
- **新增 Agent 场景**：添加 NvcAgentScene 枚举值 + DB 配置
